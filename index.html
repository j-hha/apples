What?

A set of charts built as web components. Web components are pieces of reusable code. The technology used is browser native. No additional framework has been used.

How?

Web components consist of three browser native technologies: HTML templates, custom elements and the shadow DOM. When the shadow DOM is attached to a component, it lives in its own encapsulated context. You can use the component anywhere without accidentally breaking outside styles.

This paragraph, for example, has the same css class as the chart's caption. Yet, none of the defined styles should come into effect here. The background will therefore stay white.

Gotchas

Be aware that the components might inherit some outside styles, though. To prevent style leakage here, the following style has been added to the component: :host { all: unset }. If you change the font of the body now, for example, it will not affect the bar chart component. However, if you change the font on the * selector, the style will still leak into the bar chart component.

This bar chart shows the consumption in tons for the most popular fruit varieties in Germany in the economic year 2022/2023.

Why?

No framework needed
This is all possible with vanilla JS: No JS bloat from unused framework code and resulting performance costs.
Prettier markup with templates and slots
JS and HTML become more readable.
Reusable code
The components can be easily placed in various contexts with different data.
Encapsulated components
The components live in their own context. Their logic and styles should not affect any outside elements.
